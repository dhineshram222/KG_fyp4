Sliding Window Protocols are used in the Data Link Layer and TCP to allow a sender to transmit multiple sequential frames before needing an acknowledgment, improving efficiency through pipelining. These protocols use sequence numbers to track frames and identify missing data. The two main types are Go-Back-N (GBN) and Selective Repeat (SR) ARQ. In GBN, the sender has a window size of N while the receiver has a window size of 1; if a frame or acknowledgment is lost, the sender must retransmit the entire current window starting from the missing frame. Conversely, Selective Repeat is a more advanced and efficient version used when errors are frequent. In SR, the sender retransmits only the specific missing frame rather than the whole window, which allows for faster data recovery. The sequence numbers for these frames are determined by the number of bits allocated based on the sender's window size. Ultimately, these protocols ensure reliable data transmission by managing how and when frames are repeated after a network error.
