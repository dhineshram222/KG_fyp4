A Database Management System (DBMS) comprises hardware, software, data, procedures, and users, where data acts as the vital bridge between machine and human components. Data models serve as conceptual tools to define the structure, relationships, semantics, and constraints of this data. The Relational model is the most prevalent today, organizing information into tables (relations) consisting of fixed-format records. Entity-Relationship (ER) models use schematic diagrams of real-world objects to assist in database design prior to implementation. Object-based models extend ER features with object-oriented programming concepts like inheritance to handle complex multimedia data. Semi-structured models, such as XML, provide flexibility for data transfer between applications using user-defined tags. Older structures like Hierarchical (tree-based) and Network (graph-based) models are now mostly obsolete as they were too closely tied to specific underlying implementations.
