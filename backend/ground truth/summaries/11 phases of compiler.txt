Compilers translate high-level source code into machine-readable language through six distinct sequential phases. Lexical analysis groups characters into tokens, while syntax analysis builds a parse tree to verify structural rules. Semantic analysis then ensures the code's meaningfulness through tasks like type-checking and scope resolution. An intermediate code generator produces a simplified machine-independent representation, often called Three Address Code. Code optimization refines this intermediate version to improve execution speed and minimize memory usage. The target code generator concludes the process by mapping instructions to specific machine registers and assembly language. The Symbol Table tracks identifiers throughout these stages, while the Error Handler reports any logical or syntax issues. These phases are divided into a front end (analysis) and a back end (synthesis) for platform flexibility. Standardized tools like Lex and Yacc are frequently utilized to implement these complex translation phases.
