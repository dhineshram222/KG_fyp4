Data abstraction in DBMS focuses on hiding complex implementation details while providing users with only essential information. This is achieved through three levels: the physical level, which describes how data is stored on hardware using data structures; the logical level, which defines what data is stored and their relationships; and the view level, the highest level of abstraction. The logical level provides physical data independence, ensuring that changes to the logical structure do not necessitate a physical rewrite. Additionally, the view level allows for multiple customized views and ensures security by granting different users access to specific data subsets.
