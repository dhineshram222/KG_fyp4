A relational database is a collection of tables, or relations, that store both data and the relationships between them. Each relation consists of attributes (columns) and tuples (rows), where each row represents a set of related values. The schema defines the structural design, while a relation instance refers to the data present at a specific moment. Domains specify the permitted values for attributes, which should ideally be atomic, meaning they are indivisible. Null values are used as special markers to represent data that is either unknown or does not exist. Common operations include select, insert, update, and delete, allowing users to manipulate and retrieve information. The degree of a relation is the total number of attributes, while cardinality refers to the total number of tuples. While data can be sorted for display, relations are generally unsorted within the actual database storage. This model offers simplicity and data integrity, ensuring no duplicate records exist through unique constraints. However, limitations include fixed field lengths and potential complexity when managing extremely large systems.
