I. Introduction to IP Addresses

• Definition: IP stands for Internet Protocol address; it is a unique logical identifier assigned to every device in a network.
• Function: It allows the identification of devices and their actions within the network layer (Layer 3 of the OSI model).
• Packet Role: Every network packet contains a source and destination IP address to ensure data reaches the correct node.
• Address Types: There are two primary versions: IPv4 (32-bit) and IPv6 (128-bit).

II. IPv4 (Internet Protocol Version 4)

• General Characteristics:
    ◦ Introduced in 1981, it is the most commonly used addressing system.
    ◦ It uses a 32-bit address length, which provides approximately 4.3 billion unique addresses.
    ◦ Addresses consist of four octets (8 bits each), totaling 4 bytes.
• Notation and Range:
    ◦ Represented in dotted-decimal format (e.g., 192.168.10.100) or binary format.
    ◦ Each octet ranges from 0 to 255.
    ◦ The total range spans from 0.0.0.0 to 255.255.255.255.
• Validation Rules (Spotting Errors):
    ◦ Must have exactly four octets.
    ◦ No leading zeros are permitted (e.g., 10.065.34.56 is invalid).
    ◦ Values cannot exceed 255.
    ◦ Mixing binary and decimal notations in a single address is not allowed.
• Structure: Typically divided into a Network address (identifying the specific network) and a Host address (identifying the specific device).

III. IPv6 (Internet Protocol Version 6)

• General Characteristics:
    ◦ Introduced in 1998 to resolve the address space limitations of IPv4.
    ◦ Uses a 128-bit alpha-numeric addressing system, supporting 340 trillion unique devices.
    ◦ Consists of eight segments (octets).
• Structure:
    ◦ Network Part: First 48 bits.
    ◦ Subnet ID: 16 bits used to identify sub-networks.
    ◦ Client/Host Part: Last 64 bits.
• Advantages:
    ◦ Supports real-time data transmission (useful for live streaming and satellites).
    ◦ Enhanced security through built-in encryption and decryption.
    ◦ Faster packet processing and accurate communication.
• Disadvantages:
    ◦ IPv4 and IPv6 machines cannot communicate directly with each other.
    ◦ IPv6 is often incompatible with older IPv4-capable devices.

IV. Address Categories and NAT

• Public vs. Private Addresses:
    ◦ Private IP: Used for local communication within a specific institute or home (e.g., devices connected to a single Wi-Fi router).
    ◦ Public IP: Provided by an Internet Service Provider (ISP) for global internet communication.
• Network Address Translation (NAT):
    ◦ A technology used to map multiple private IP addresses to a single unique public IP.
    ◦ NAT is the primary reason why billions of devices can operate despite the 4-billion address limit of IPv4.
• Uniqueness: Two devices on the internet can never have the same public address at the same time.
