I. Fundamental Definitions

• Software: A collection of programs designed for specific tasks (e.g., browsers, operating systems).
• Database: A collection of interrelated data stored for various operations.
• DBMS: A software program or system used to manage, store, and retrieve data in an efficient and centralized manner.

II. Real-World Applications of DBMS

A DBMS is utilized in nearly every modern field, including:
• Banking: Managing deposits and transactions.
• Travel & Hospitality: Hotel and airline reservation systems.
• E-commerce: Tracking online shopping items and customer data.
• Education: Managing student records and library systems.
• Real-time Systems: Weather forecasting and Geographical Information Systems (GIS) which analyze satellite images and maps.

III. Limitations of File Systems vs. DBMS Advantages

The sources identify seven major disadvantages of file systems that a DBMS effectively resolves:
1. Data Redundancy and Inconsistency
• File System: Different programmers use different file structures, leading to duplicated data (redundancy) in multiple locations,. This increases storage costs and leads to inconsistency, where a change in one file (e.g., a student's address) does not automatically update in another.
• DBMS: Minimizes duplication and ensures that a change made in one place is automatically reflected across all associated records.
2. Difficulty in Accessing Data
• File System: Retrieving specific information (e.g., employees with a salary over $50,000) requires manual effort or complex custom programming for every new request.
• DBMS: Provides a more responsive system where users can easily retrieve data using queries based on specific conditions.
3. Data Isolation
• File System: Data is often scattered in different files and formats, making it difficult to extract relevant information.
• DBMS: Stores all records in a centralized repository, making data isolation and retrieval straightforward.
4. Integrity Problems
• File System: It is difficult to enforce consistency constraints (e.g., ensuring a salary is never $0 or a phone number is exactly 10 digits).
• DBMS: Allows administrators to apply constraints directly to the data; any update that violates these rules is blocked.
5. Atomicity Problems
• File System: If a system fails during a multi-step process (like a bank transfer), it may result in partial execution (e.g., money is debited from one account but not credited to the other).
• DBMS: Guarantees atomicity, meaning a transaction must be "all or none." If a failure occurs, the system revokes the partial changes to maintain a consistent state.
6. Concurrent Access Anomalies
• File System: When multiple users access the same file simultaneously, it can lead to anomalies. For example, two clerks might read the same balance and update it incorrectly, losing data from one of the transactions.
• DBMS: Manages concurrent access to allow multiple users to interact with the same data without creating errors or inconsistencies.
7. Security Problems
• File System: While a file can be password-protected, it is difficult to give a user access to a file while hiding specific parts of it (like a salary column).
• DBMS: The Database Administrator (DBA) can grant specific privileges and "views," ensuring users only see the data they are authorized to see.
