I. Overview of the OSI Model

• Definition: Developed by the ISO (International Organization for Standardization) in 1984, the OSI model is a seven-layer framework designed to ensure accurate and smooth communication between network devices.
• Layer Groupings:
    ◦ Hardware Layers (1–3): Physical, Data Link, and Network layers.
    ◦ Software Layers (5–7): Session, Presentation, and Application layers.
    ◦ Heart of OSI (Layer 4): The Transport Layer acts as the bridge connecting hardware and software layers.
• Communication Flow: A sender processes data from Layer 7 down to Layer 1, while the receiver processes data from Layer 1 up to Layer 7.

II. Layer Breakdown and Functionalities

1. Application Layer (Layer 7)
• Role: Directly interacts with the user and software applications.
• Services: Provides file transfer, mail services, and directory services.
• Key Protocols: HTTP (web browsing), FTP (file transfer), SMTP (email), and Telnet (virtual terminals).
2. Presentation Layer (Layer 6)
• Role: Concerned with the syntax and semantics of the information exchanged.
• Functions:
    ◦ Translation: Converts data into machine-understandable formats (e.g., binary).
    ◦ Data Compression: Reduces the number of bits to speed up transmission, especially for multimedia.
    ◦ Encryption/Decryption: Ensures privacy by converting original messages into unreadable formats and back.
3. Session Layer (Layer 5)
• Role: Establishes, maintains, and synchronizes interactions between devices.
• Functions:
    ◦ Session Management: Uses protocols like NetBIOS to manage connections.
    ◦ Authentication & Authorization: Verifies user identity (e.g., usernames/passwords) and checks permissions to access specific resources.
4. Transport Layer (Layer 4)
• Role: Responsible for process-to-process delivery of the entire message.
• Functions:
    ◦ Segmentation & Reassembly: Breaks large data into smaller units (segments) with sequence numbers for reassembly.
    ◦ Port Addressing: Uses port numbers (e.g., HTTP = 80, FTP = 20) to ensure data reaches the correct application.
    ◦ Error & Flow Control: Ensures reliable transmission through speed matching (flow control) and retransmitting lost/corrupted data.
• Main Protocols:
    ◦ TCP: Connection-oriented, reliable, and sequencing-based.
    ◦ UDP: Connectionless, faster, but lacks feedback/reliability (used for streaming/gaming).
5. Network Layer (Layer 3)
• Role: Manages the delivery of data (called packets) between different networks.
• Functions:
    ◦ Logical Addressing: Assigns unique IP addresses to senders and receivers.
    ◦ Routing & Path Determination: Finds the best/shortest physical path using protocols like OSPF or BGP.
6. Data Link Layer (Layer 2)
• Role: Responsible for moving data (frames) from one node to another.
• Functions:
    ◦ Framing: Adds headers and trailers to packets.
    ◦ Physical Addressing: Uses MAC addresses (12-digit hardware addresses) to identify devices on a local network.
    ◦ Access Control: Determines which device has control over a shared communication link to prevent collisions.
7. Physical Layer (Layer 1)
• Role: Transmits raw bits over physical media (wired or wireless).
• Key Aspects:
    ◦ Media Representation: Converts bits into electrical signals (Ethernet), light signals (Fiber), or radio waves (Wireless).
    ◦ Specifications: Defines data rate, bit synchronization, and physical topology (e.g., star, ring, mesh).
    ◦ Transmission Mode: Decides between Simplex, Half-duplex, or Full-duplex communication.
