I. Definition and Core Architecture

• Linked List: A linear data structure consisting of a group of elements called nodes.
• Node Components: Every node is divided into two parts:
    ◦ Data Part: Stores the actual information or value.
    ◦ Next (Link) Part: Stores the memory address of the following node, allowing them to connect in a chain-like structure.
• Non-Contiguous Memory: Unlike arrays, linked list elements are not stored in contiguous memory locations; they are allocated at runtime.

II. Essential Terminology

• Head: A specific pointer that always points to the first node of the list; it is the entry point for accessing the data.
• Pointer: Used to navigate between nodes and identify the start and end of the sequence.
• Null Pointer: A value in the "next" part of the last node indicating there is no succeeding node.
• Traversing: The process of moving from one node to another using addresses to access data.

III. Comparison: Arrays vs. Linked Lists

• Size and Resizing: Arrays have a fixed size and resizing them is expensive, whereas linked lists have a dynamic size that can grow or shrink during program execution.
• Insertion and Deletion: These operations are inefficient in arrays because elements must be shifted. In linked lists, they are efficient because only the pointers need to be updated.
• Access Speed: Arrays allow for random access using indices (O(1) time). Linked lists only support sequential access, meaning you must travel from the head to reach a specific node (O(n) time).
• Memory Usage: Arrays may waste memory if not full, but linked lists require extra memory to store the pointers for each node.

IV. Types of Linked Lists

• Simple (Singly) Linked List: Navigation is restricted to forward-only.
• Doubly Linked List: Nodes contain pointers to both the previous and next nodes, allowing for bidirectional navigation and backtracking.
• Circular Linked List: The last element contains a link back to the first element, forming a continuous loop.

V. Common Operations

• Insertion: Adding new elements at the beginning, end, or middle of the list.
• Deletion: Removing nodes by key or specific position.
• Search: Finding a specific element within the list using a given key.
• Display: Traversing the entire list to show all stored values.

VI. Advantages and Disadvantages

• Advantages:
    ◦ No need to define an initial size.
    ◦ Easily implements other structures like stacks and queues.
    ◦ Flexible insertion/deletion at any point.
• Disadvantages:
    ◦ Requires extra storage for pointers.
    ◦ Time-consuming traversal to access individual nodes.
    ◦ Reverse traversal is difficult or impossible in singly linked lists.

VII. Real-World Applications

• Software Functions: Implementing undo/redo operations in programs like Word or Photoshop.
• Web Browsers: Managing history and navigation (back/forward buttons).
• System Organization: Managing lists of file names within folders.
• Modeling Services: Representing lines, such as a cashier line in a bank or airport.
• Data Structures: Serving as the foundation for hash tables and binary trees.
