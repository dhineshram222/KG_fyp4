I. Overview of the ER Model

• Definition: ER stands for Entity-Relationship model, a high-level conceptual data model used to represent the relationships between elements in a database.
• Purpose as a Blueprint: It acts as a design plan or blueprint, similar to a civil engineer's drawing, allowing designers to understand the overall logical structure of a database before implementation.
• Design Phase: It is developed during the conceptual design phase to specify the requirements of an Enterprise schema.

II. Core Components

The ER model consists of three primary components: Entities, Attributes, and Relationships.
1. Entities and Entity Sets
• Entity: A real-world object, person, place, or thing that is distinguishable from others (e.g., a specific student or instructor).
• Entity Set: A collection of entities of the same type that share the same properties (e.g., the set of all instructors at a university).
• Types of Entities:
    ◦ Strong Entity: Independent and possesses a primary key to uniquely identify it.
    ◦ Weak Entity: Completely dependent on a strong entity and lacks a primary key; it uses a discriminator key instead.
2. Attributes
Attributes describe the properties or characteristics of an entity.
• Key Attribute: Represents the primary key used to uniquely identify an entity (e.g., Student ID).
• Composite Attribute: An attribute that can be divided into sub-attributes (e.g., a "Name" attribute split into First, Middle, and Last name).
• Multi-valued Attribute: An attribute that can have more than one value for a single entity (e.g., multiple phone numbers).
• Derived Attribute: An attribute whose value is calculated from other attributes (e.g., calculating "Age" from a "Date of Birth").
3. Relationships and Relationship Sets
• Relationship: An association among several entities (e.g., "Instructor advises Student").
• Relationship Set: A set of relationships of the same type.
• Mapping Cardinalities (Types):
    ◦ One-to-One (1:1): One instance on the left relates to only one on the right (e.g., one female married to one male).
    ◦ One-to-Many (1:M): One instance on the left relates to multiple on the right (e.g., one scientist invents many inventions).
    ◦ Many-to-One (M:1): Multiple instances on the left relate to one on the right (e.g., many students enroll in one course).
    ◦ Many-to-Many (M:M): Multiple instances on both sides relate to each other (e.g., many employees assigned to many projects).

III. Visual Representation (ER Diagrams)

The ER model is portrayed visually using specific shapes to represent different components:
• Rectangles: Represent Entities (Double rectangles represent Weak Entities).
• Ellipses: Represent Attributes.
    ◦ Underlined Text in Ellipse: Key Attribute.
    ◦ Double Oval: Multi-valued Attribute.
    ◦ Dashed Ellipse: Derived Attribute.
• Diamonds (Rhombus): Represent Relationships.
