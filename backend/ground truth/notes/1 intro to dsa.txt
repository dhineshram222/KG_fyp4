I. Fundamentals of Data and Information

• Data: A collection of different types of information, such as numbers, alphabets, characters, or symbols upon which a computer performs operations. It can be thought of as quantities or values stored on magnetic or optical media.
• Information: Raw data that has been processed and arranged in a systematic way so that it becomes meaningful and understandable to the user.
• Data Object: A unique location in computer memory that holds a value and is identified by a unique name within a program.
• The Program Formula: A computer program is essentially the combination of algorithms and data structures.

II. Definition and Purpose of Data Structures

• Definition: A data structure is a systematic way of organizing, managing, and storing data in a computer so it can be used efficiently. It represents the logical or mathematical relationship between data elements.
• Primary Goals:
    ◦ Efficiency: To optimize performance in terms of time (execution speed) and space (memory usage).
    ◦ Complexity Management: To provide fast solutions for increasingly complex human requirements and large-scale data.
    ◦ Resource Optimization: Using structures like Arrays allows a programmer to store 100 values in a single structure rather than creating 100 individual variables.

III. Classification of Data Structures

Data structures are classified based on how they are built and how they organize data:
1. Primitive vs. Non-Primitive
• Primitive: Standard, predefined, built-in data types provided by programming languages (e.g., Integer, Float, Character, Double, and Pointer).
• Non-Primitive: Derived from primitive structures; they focus on grouping homogeneous (same type) or heterogeneous (different type) data items.
2. Linear vs. Non-Linear
• Linear: Elements are stored in a sequential or linear order (e.g., Arrays, Stacks, Queues, Linked Lists).
• Non-Linear: Data is stored in a random or hierarchical manner where one item can connect to several others (e.g., Trees and Graphs). These often represent parent-child or hierarchical relationships.
3. Static vs. Dynamic
• Static: Has a fixed size that is defined at compile time and cannot be changed during execution (e.g., Arrays).
• Dynamic: The size is not fixed; the structure can expand or shrink during execution to manage memory more flexibly (e.g., Linked Lists).

IV. Common Operations on Data Structures

Data structures are designed to support several core operations:
• Traversing: Accessing every data element exactly once to process it.
• Insertion: Adding a new data element to the structure.
• Deletion: Removing an existing element from the structure.
• Searching: Finding the location of a specific value.
• Sorting: Arranging elements in a specific logical order (ascending or descending).
• Merging: Combining two different files or structures into a single one.

V. Real-World Applications

The importance of data structures is demonstrated through their use in daily technology:
• Stacks (Undo/Redo): Applications like Google Docs and Microsoft Word use stacks to track user actions. When you perform an action, it is "pushed" onto an Undo stack; using "Control+Z" "pops" the top element off and moves it to a Redo stack.
• Arrays (Bitmap Images): Digital images are stored as bitmaps, which are essentially two-dimensional arrays of pixels. Each cell in the array stores color information for a specific "square" of the image.
• Graphs (Social Networks): Platforms like Facebook use graphs to map friendships. Each person is a "node," and the connection between friends is represented by an "edge".
