LECTURE NOTES
=============


1. Compiler
Definition: A translator that converts high-level programming language into machine language, analyzes the complete source program, compiles it, and returns all errors to users.
  - High-level Programming Language: Language written by the developer or user (e.g., C, C++, Java), generally in English language.
  - Machine Language: Binary language, generally in 0101 format, understood by the processor, also called target code or binary language.
  - Source Program: The initial programming code provided by a developer that is processed by the compiler.
  - Errors: Issues or mistakes detected during the compilation process, categorized by the phase in which they occur, such as wrong spelling tokens, wrong syntax, mismatched operands, or undeclared identifiers.

5. Target Program

6. Phases Of Compiler
Definition: A total of six sequential steps (Lexical Analyzer, Syntax Analyzer, Semantic Analyzer, Intermediate Code Generator, Code Optimizer, and Code Generator) used to convert a source program into a target program.
  - Lexical Analyzer: The first phase of the compiler that takes source code as input, reads it character by character from left to right, and converts it into meaningful lexemes or tokens.
  - Syntax Analyzer: The second phase of the compiler, also called parsing, that takes tokens as input, checks the syntax in the program as per programming language rules, and generates a parse tree.
  - Semantic Analyzer: The third phase of the compiler that takes syntax tree and symbol table as input, checks the meaning of each and every line in the program, and tracks identifiers, their types, and expressions.
  - Intermediate Code Generator: The fourth phase of the compiler that generates an intermediate representation of the source program, also called three-address code, which is between high-level and machine-level language and is easy to translate.
  - Code Optimizer: The fifth and optional phase of the compiler that optimizes the intermediate code by removing unnecessary lines and unused variables for faster execution and less memory space.
  - Code Generation: The sixth and final phase of the compiler that takes optimized intermediate code as input, translates it into machine code, and allocates proper memory space and registers to the program.
  - Target Program: The machine language output resulting from the compilation of a source program.

7. Lexemes
Definition: Meaningful sequences of characters derived from the source code by the lexical analyzer.
  - Tokens: Collections of lexemes, which represent identified elements like operators, identifiers, or constants in the source program.

10. Parse Tree
Definition: A hierarchical representation generated by the syntax analysis phase to check for syntax errors.
  - Terminals: The set of elements in a grammar like identifiers, operators, and semicolons.
  - Variables / Non-terminals: The set of elements in a grammar like S, E, T, F, which can be rewritten.
  - Yield of the Parse Tree: Found by traversing the parse tree top to bottom, left to right, taking notes of only the terminals.
  - Start Symbol (S): A non-terminal symbol in production rules that can be rewritten to derive expressions.
  - Semantic Analysis Phase: The third phase of the compiler where the semantic analyzer performs logical analysis.

16. Compiler Phases
Definition: The different stages of the compiler.

17. Language Translator
Definition: Needed to convert human readable source code into machine code.
17.1 Compiler (Specific)
A phase of the language translator that converts pure high-level language code into equivalent assembly language code.
  - Lexical Analysis Phase: The first of the six phases of the compiler where the lexical analyzer processes input.
  - Syntax Analysis Phase: The second phase of the compiler where the syntax analyzer processes a stream of tokens.
  - Semantic Analysis Phase: The third phase of the compiler where the semantic analyzer performs logical analysis.
  - Intermediate Code Generation Phase: The fourth phase of the compiler where intermediate code is produced.
  - Code Optimization Phase: The fifth phase of the compiler where intermediate code is optimized.
  - Target Code Generation Phase: The sixth and final phase of the compiler, where the final assembly code is produced.
  - Pre-processor: A phase of the language translator that converts high-level language code into pure high-level language code.
  - Assembler: A phase of the language translator, following the compiler.
  - Linker and Loader: The final phase of the language translator.
  - Human Readable Source Code: The input to a language translator.
  - Machine Code: The output of a language translator.

26. Syntax Analyzer / Parser
Definition: The component in control of the syntax analysis phase, analyzing the stream of tokens following specific production rules.
  - Type 2 / Context-Free Grammars (CFG): Grammars that the syntax analyzer depends on.
  - Production Rules: Specific set of rules used by the parser to form the parse tree.
  - Syntax Error: An error detected if the yield of the parse tree and the provided stream of tokens are not the same.

36. Intermediate Code
Definition: The output of the intermediate code generator, serving as a machine-independent representation of the code.
  - Three-Address Code (TAC): A popular intermediate code representation where statements have at most three addresses.
  - Code Optimization Phase: The fifth phase of the compiler where intermediate code is optimized.

37. Three-Address Code (Tac)
Definition: A popular intermediate code representation where statements have at most three addresses.
  - Address: Refers to the addresses of variables within three-address code statements.

38. Temporary Variable
Definition: Variables (e.g., T0, T1) created by the intermediate code generator to store intermediate results.

39. Address
Definition: Refers to the addresses of variables within three-address code statements.

40. Frontend
Definition: The collective term for the first four phases of the compiler (Lexical Analysis, Syntax Analysis, Semantic Analysis, Intermediate Code Generation).
  - Lexical Analysis Phase: The first of the six phases of the compiler where the lexical analyzer processes input.
  - Syntax Analysis Phase: The second phase of the compiler where the syntax analyzer processes a stream of tokens.
  - Semantic Analysis Phase: The third phase of the compiler where the semantic analyzer performs logical analysis.
  - Intermediate Code Generation Phase: The fourth phase of the compiler where intermediate code is produced.

41. Optimized Code
Definition: The output of the code optimizer, a reduced or improved version of the intermediate code.
  - Target Code Generation Phase: The sixth and final phase of the compiler, where the final assembly code is produced.

42. Machine Dependent Optimization
Definition: A type of code optimization that depends on the target machine architecture.

43. Machine Independent Optimization
Definition: A type of code optimization that is not specific to any particular machine architecture.

44. Backend
Definition: The collective term for the last two phases of the compiler (Code Optimization and Target Code Generation).
  - Code Optimization Phase: The fifth phase of the compiler where intermediate code is optimized.
  - Target Code Generation Phase: The sixth and final phase of the compiler, where the final assembly code is produced.

45. Target Code Generator
Definition: The component that takes optimized code and finally produces the assembly code segment.

46. Assembly Code Segment
Definition: The final machine-specific output of the target code generator.
  - Mnemonic: Instructions used in assembly code (e.g., MOV, IMUL).
  - Registers (EAX, EDX): CPU components like EAX and EDX used to store data during assembly code execution.
  - D Word PTR: A data word pointer used in assembly code.
  - RBP (Register-Based Pointer): A register-based pointer used in assembly code.
