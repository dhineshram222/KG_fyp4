In this presentation we will focus on the view of data. Why do we need multiple views of data? We need to understand what is data abstraction. In order to understand data abstraction that is hiding the complexity let's take an analogy for better understanding. Let's start with the first level the physical level. This physical level is actually the lowest level of abstraction. This level deals with how the data are stored. This storage may be involving simple data structures or even complex data structures. So we need not rewrite the physical structure if there is a change in the logical structure. We need not show that complexity to the user and that's why this level deals with the users and the access to the users.
